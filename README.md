# ResQ Hub

> 專案名稱暫定。

台灣災害協作的知識庫與數位工具——從整理第一手經驗出發，抽象成可複製的協作模式，最終發展成實用工具。

---

## 起源

2025 年花蓮馬太鞍溪堰塞湖溢流救災。

民間志工動員能量極強，但資訊流極度分散：需求重複回報、志工重複出動、物資去向難追蹤、事後沒有可回顧的紀錄。

問題不是工具不夠，而是缺乏可被複製的協作模式與資訊架構。

這個專案想解決的核心問題：

> **「我現在過去，是否真的還幫得上忙？」**

---

## 三層架構

| 層次 | 目標 |
|------|------|
| **經驗層** | 保留完整脈絡——誰看到了什麼、當下手上有哪些資訊、為什麼會誤判 |
| **模式層** | 從多個經驗抽象出可複製的原則與角色分工 |
| **工具層** | 災時降低資訊落差的數位工具 |

這三層也是開發順序，先把經驗留下來，才有條件做工具。

## 目前狀態

🔨 **建構中** — 知識庫架構已建立，正在整理第一手現場經驗與研究資料。
