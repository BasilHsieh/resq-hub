# 問題地圖

> **用途**：問題優先級排序、v1 場景選擇的依據。
> **核心觀察**：以下 10 個問題大多 14-26 年未解。**這不代表 ResQ Hub 容易做，代表這些問題很硬**——但也代表民間替代方案的空間真實存在。

## P1. 物資統合失靈

### 證據鏈

- **921 (1999)**：南投縣長透過收音機呼籲物資 → 3 天物資塞滿體育館 → 又呼籲「物資已足、改捐金錢」（楊永年 2009 紀錄）
- **88 風災 (2009)**：「貨櫃整批整批運下來，它們都停在高速公路上（水太深）」（屏東社工 A1）；二手衣送到災區被退回（媒體報導 2009/08/17）
- **林萬億 2025**：花蓮光復現場「物資爆量 8 大原因」、「物資倉儲缺資訊系統、不更新登錄」
- **數位化平台論文 2019**：NCDR 計畫提到「失蹤人口、防救災物資分配、志工資訊」是政府忽略需要 VGI 補的領域

### 為何沒解

- 物資募集是民間 NGO 的主要工作，但分配是地方政府的責任 → 中間沒人統合
- 媒體呼籲 → 群眾捐贈 → 物資湧入 是無法控制的反射弧
- 任何「物資 dashboard」都需要 NGO 揭露庫存——這違反募款競爭邏輯（P7）

### ResQ Hub 切入點

- ⭐ 不是做物資管理系統，是做**物資需求即時可見化**——讓捐贈者出發前知道現場最缺什麼/已飽和
- 避免做需要 NGO 揭露全部庫存的功能（會失敗）
- 可以是「現場節點各自回報缺口」式的去中心化資訊聚合

## P2. 災情回報過載

### 證據鏈

- **數位化平台論文 2019**：新北市消防局原話「災害來時都快忙不過來了，如果全部的回報系統都要上傳實在太多了…如果能夠一次上傳到所有要資料的雲端，那就會比較有效率一點」
- **論文觀察**：政府內部已有多個系統 + LINE 群組，第一線「重複登打」「無餘力通報實際災情」
- **強韌台灣計畫 2022**：縱向溝通已落實，水平協力仍須建立

### 為何沒解

- 每個部會/層級/縣市都各自開發系統 → 整合阻力來自既有系統的沉沒成本
- NCDR 計畫 (2015-) 嘗試整合，但只整合到政府/學研層，民間端排除在外
- 沒有人有權強迫所有單位停用自家系統改用統一系統

### ResQ Hub 切入點

- ⭐ 不要再加一個系統！目標應該是**整合既有系統（特別是 LINE）的盲點**
- 例如：跨 LINE 群組的關鍵字搜尋、訊息結構化抽取、重要訊息推送
- 不要要求第一線「再登一次資料」——這是 P2 的核心錯誤

## P3. LINE 群組混亂

### 證據鏈

- **數位化平台論文 2019**：地方政府內部各層級皆有 LINE 群組做災情回報；連 NCDR 計畫辦公室都透過 LINE 推廣公民上傳情資
- **屏東八八社工 2012**：A3「我自己也跟裡面的人保持溝通的管道…知道裡面還缺什麼」——「溝通的管道」就是 LINE/電話
- **林萬億 2025**：花蓮光復現場大量 LINE 群組

### 為何沒解

- LINE 是事實上的台灣全民通訊工具，**無法跳過**
- 群組數量隨需要無限增生，無人統管
- 跨群同步靠人工複製貼上

### ResQ Hub 切入點

- ⭐ 不要試圖取代 LINE
- ⭐ 補 LINE 的盲點：
  - 跨群搜尋（同一資訊散落 10 個群，找不到）
  - 資訊結構化（從一片文字流抽出「物資需求」「志工需求」「狀況回報」等結構）
  - 重要訊息提醒（被洗版的關鍵需求）
  - 跨群同步（一個訊息要轉到 5 個群，自動化）

## P4. 組織間互踢責任

### 證據鏈

- **88 風災 (2009)**：「中央政府與地方政府溝通不良，將問題互踢皮球」（鄭阡妤 2014）；中央槓屏縣府互嗆救災不力（2009/08/09 自由時報）；台南市長許添財公開批國軍救災緩慢（2009/08/08）
- **楊永年 2009**：「中央與地方政府合作機制的建立也相當重要」「有兩個部會派人協助救災，但其中一位部會的官員卻經常更換」
- **林萬億 2025**：花蓮光復橫向協調缺口
- **數位化平台論文 2019**：CCTV 案例——消防局要架接警政單位 CCTV 受阻，要拉到災防辦會報層級協調

### 為何沒解

- 法律已給地方首長動員國軍權限，但實作仍失敗——**法規不缺、誘因才缺**（楊永年明確指出）
- 政治化（藍綠對立）讓中央地方無法純就專業合作
- 各部會「本位主義」沒有打破的機制

### ResQ Hub 切入點

- ⭐ 不可能解組織政治問題
- 但可以**讓組織間資訊透明化**——當所有人都看到同一份現場資訊，互踢責任的成本變高
- 風險：公開資訊也可能被武器化（變成媒體攻擊材料）

## P5. 公所量能不足

### 證據鏈

- **楊永年 2009**：「鄉公所在救災所發揮的功能相當有限，反而是消防與警察發揮救災功能」
- **林萬億 2025**：花蓮光復公所自己受災，無法擔任強韌台灣計畫所設的「民間協作樞紐」
- **謝志強 2014**：行政院災害防救辦公室全國僅 25 人編制
- **數位化平台論文 2019**：第一線「忙不過來」的原話

### 為何沒解

- 結構性編制問題——公所平時就人少，災時不會突然增援
- 防災非主業，沒有專責人員
- 偏遠地區公所本身就是受災戶

### ResQ Hub 切入點

- ⭐ 預設**沒有公所對接** = 設計時不能依賴政府窗口
- 民間 NGO/志工要能繞過公所自行協作
- 但要尊重公所還在運作的部分（不要踩線）

## P6. 災變教育空白

### 證據鏈

- **屏東八八社工 2012**：「之前都沒有…在這方面也是我覺得其實平時應該都要加強的」（A3）
- **論文作者整理**：「面對突如其來的災難，危機處理在實務上不曾運用過的訓練，必須要開始使用在救災的任務中，但是莫拉克風災是第一次，造成社工員們仍然是手忙腳亂」
- **強韌台灣計畫 2022** 試圖以「防災士」填補，但**規模和滲透度仍小**（林萬億 2025 觀察）

### 為何沒解

- 防災教育需要長期投入但短期看不到效果
- 教育成本 vs 「不一定會發生」的猶豫
- 流動率高（P9）讓訓練效果衰減

### ResQ Hub 切入點

- ⭐ 工具設計可以**繞過訓練要求**——「不需要訓練就能用」是 P6 的解答
- 例如 Xdite 2009 莫拉克災情支援網：無人需要訓練就能貼/讀/留言
- 設計上避免複雜流程、術語、SOP 要求

## P7. NGO 募款競爭

### 證據鏈

- **黃盈豪 2010**：直接點名莫拉克時「各社福團體爭相對外募款瓜分資源」、「濫發現金補助影響安置」是「天災後的人禍」
- **慈濟莫拉克 2013**：揭露宗教文化與外部訴求的張力
- **林萬億 2025**：花蓮光復多個 NGO 各自運作

### 為何沒解

- 募款是 NGO 生存命脈，揭露現場全貌等於部分讓出競爭優勢
- 沒有強制協調機制
- 公部門無權干預民間募款分配

### ResQ Hub 切入點

- ⭐ 不要解這個問題——超出工具能力範圍
- 設計時要避免**強化**競爭：不要做「現場 NGO leaderboard」、「物資分配表」這類會引發爭議的功能
- 可以做「中性的需求公告板」——任何 NGO 都能讀，但揭露自家資訊是 opt-in

## P8. 媒體填補資訊真空 → 失敗模式

### 證據鏈

- **鄭阡妤 2014**：921「政府救災系統有心無力、媒體指揮救災、民間自救」；88「中央救災中心災情資訊相當程度仰賴電視台架設之『災情民眾上傳影片』」
- **屏東八八社工 2012**：媒體壓力 → 違反專業判斷 → 救難直升機失事（2009/08/09，全機罹難）
- **鄭阡妤 2014**：「依賴媒體傳播的資情資訊不見得能夠專業、全面與完整，甚至可能出現誇大問題」

### 為何沒解

- 媒體競爭機制 → 即時性 + 戲劇性優先於準確性
- 自媒體普及後問題擴大
- 沒有競爭對手 → 媒體就是事實上的資訊樞紐

### ResQ Hub 切入點

- ⭐⭐ 如果 ResQ Hub 成為民間信賴的資訊源，可以**分擔媒體目前扛的角色**
- 但要意識到：呈現方式不是中性的。例如「為何政府沒救」的視覺化會塑造輿論壓力
- 設計上應該偏向**事實性資訊**（缺什麼、需要什麼），避免**判斷性資訊**（誰沒做事）

## P9. 公務員流動率高 → 培訓白做

### 證據鏈

- **楊永年 2009**：「有兩個部會派人協助救災，但其中一位部會的官員卻經常更換，導致溝通連繫的困難」
- **鄭阡妤 2014**：地方政府指揮層級無救災專業
- **數位化平台論文 2019**：蔡孟涵等 (2014)「防救災人員異動頻繁，各局處未設防災專責組別」

### 為何沒解

- 政府人事制度問題，非防災領域能解
- 防災非主業，輪調是常態

### ResQ Hub 切入點

- ⭐ 工具設計**第一次見就能用**是必要條件
- 任何「需要交接的密技」「需要訓練的功能」會被流動消化掉
- 偏好「自我解釋的介面」「對話式輸入」等低學習成本設計

## P10. 中央依法不能接管

### 證據鏈

- **謝志強 2014**：「依據我國『災害防救法』規定，在災害應變階段，中央對於地方發生之災害，不管災害程度如何，僅有支援協調之功能…無法源依據於必要時介入整體災害應變作為」
- 對比美國總統可宣布凍結州長職權、日本內閣可指示地方
- **林萬億 2025**：花蓮光復公所自己受災無法擔樞紐 + 中央依法不能接管 = **權力真空**

### 為何沒解

- 修法是政治決定
- 涉及中央地方權力分配
- 短期難改變

### ResQ Hub 切入點

- ⭐ 這個權力真空就是民間補位的法理空間
- ResQ Hub 不是補政府的位置，是補政府**法理上不能填的位置**
- 這也是 ResQ Hub 的「為什麼是民間」的法理依據

## 跨問題觀察

### 為什麼這些問題沒解

1. **體制慣性**：既有系統的沉沒成本（P2、P5）
2. **政治化**：藍綠/中央地方角力（P4、P10）
3. **NGO 競爭**：揭露 vs 募款（P7）
4. **無人有權強制**：沒有 entity 能命令所有人改變（P1、P2、P3、P4、P7）
5. **教育成本**：長期投入短期無感（P6、P9）

### 為什麼民間工具有機會

- **不需要任何單位授權**（不像政府要修法、要跨部會協調）
- **可以 opt-in 設計**（不強迫，但提供誘因）
- **可以快速迭代**（不需要 3 年計畫週期）
- **可以重啟**（失敗了重來，不像政府要承擔政策失敗成本）

### v1 場景選擇的暗示

從問題地圖看，**ResQ Hub v1 應該對焦 P1+P2+P3**（物資/災情回報/LINE 整合），因為：
- 直接相關度最高
- 民間切入空間最大
- 跟花蓮光復現場最對得起來

P4-P10 是長期願景或無法解的，先放下。
