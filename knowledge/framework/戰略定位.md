# 戰略定位

> **這份文件是「研究綜合 → 策略建議」的橋樑**。它不是最終 vision，而是給 Basil 重寫 vision 時的參考結論。
> **Claude 的角色**：把 13 份研究歸納成可決策的選項。
> **Basil 的角色**：用花蓮現場經驗挑戰、修改、決策。

---

## 一句話定位

> **ResQ Hub 是台灣災後民間救援生態的協作層——服務散兵志工、小型 NGO、受災民眾之間的橫向協作，補政府/大型 NGO 體系不覆蓋且法理上不能覆蓋的空白。**

拆解：

| 關鍵詞 | 為什麼 |
|---|---|
| **災後**（非災時） | 既有方案重災時，災後協作期幾乎無方案 |
| **民間救援生態** | 政府版（NCDR/強韌台灣）已存在 |
| **協作層** | 不是新平台，是補既有工具（LINE 等）的盲點 |
| **散兵志工、小型 NGO、受災民眾** | 三類最未被服務、互相需要彼此 |
| **橫向協作** | 縱向協調是政府的事；橫向是民間自治的事 |
| **政府法理上不能覆蓋** | 謝志強 2014：中央依法不能接管地方應變，民間補位有法理空間 |

---

## 該做的 ✅ vs 不該做的 ❌

### ✅ 該做

| 項目 | 為什麼 |
|---|---|
| 散兵志工 + 小型 NGO 的協作工具 | 既有方案地圖顯示這層完全空白 |
| 補 LINE 的盲點（搜尋/結構化/跨群同步） | LINE 是事實標準不可取代，但有盲點 |
| 物資需求即時可見化 | 26 年沒解的 P1 物資統合失靈 |
| 災後協作期支援（週/月為單位） | 既有方案重災時，災後是空白 |
| 零門檻入口（讓散兵志工 5 秒上手） | D1+D2 必守原則 |
| 為「下一個 Xdite」提供可複用工具 | Xdite 2009 證明可行但沒留下產品 |
| 災民/部落作為節點而非接受對象 | D6 必守 |

### ❌ 不該做

| 項目 | 為什麼 |
|---|---|
| 重做 NCDR 政府版資訊平台 | 政府已有，民間做沒用 |
| 主打「給政府決策的情資」 | NCDR 的角色 |
| 「公民災情回報 → 政府」這條線 | 政府已設定排除（假新聞風險） |
| 重做防災士培訓系統 | 強韌台灣計畫的角色 |
| 跟慈濟競爭 | 規模不對等，慈濟有自己的系統與文化 |
| 大規模 NGO 之間的整合（自上而下） | NGO 競爭關係讓自上而下無解 |
| 災時即時決策支援 | 風險高、責任歸屬難、且 NCDR 已涵蓋這層 |
| 強制揭露全部資訊 | D5 違反 |
| 預設災民是被服務對象 | D6 違反 |
| 重複 Xdite 2009 已踩的坑（強制地圖、人工篩選） | 浪費時間 |

### ⚠️ 可探索但小心

| 項目 | 風險 |
|---|---|
| 跟學研機構合作 | 他們屬於 NCDR 體系，可能與民間定位有張力 |
| 跟政府單位接觸 | 易陷入政治化、被收編風險 |
| 公開呈現政府失能 | 易被媒體武器化 (D9) |
| 部落區域特殊功能 | 需要部落諮詢與共同設計 (D14) |

## v1 場景選擇建議

### 選項

| 選項 | 內涵 | 評估 |
|---|---|---|
| A. 颱風（年年發生） | 從南部颱風水災案例著手 | 機會多但無下次明確時點 |
| B. 地震（大震週期長） | 從可能的下次大震準備 | 戰備價值但測試難 |
| **C. 災後協作期（任何災種）** | 不對焦特定災種，對焦災後 1-30 天的協作 | **建議** |
| D. 既有災害（花蓮光復延伸） | 直接服務當前的花蓮重建 | 真實但需要 Basil 現場資源 |

### 推薦：**C + D 結合**

**核心戰略**：先做「災後協作期」通用工具，**第一個應用場景是花蓮光復**。

理由：
- C 提供長期方向（不依賴特定災種）
- D 提供真實場景與用戶（Basil 有現場資源）
- 兩者結合：花蓮光復的工具是 v1，做完後可複製到下次災害

## v1 用戶選擇建議

### 選項回顧（從使用者地圖）

| 選項 | 規模 | 對焦度 |
|---|---|---|
| A. 散兵志工為主 | 大（湧入時數百-數千） | ⭐⭐⭐ |
| B. 小型 NGO 為主 | 中（數十個組織） | ⭐⭐⭐ |
| C. 打雜社工為主 | 小（每縣市 20-100 人） | ⭐⭐ |
| D. 公所人員 | 小，且體制限制 | ⭐ |
| E. 受災民眾/部落 | 大但敏感 | ⭐⭐ |

### 推薦：**A + B 為主，C 為橋樑**

**v1 用戶三角**：
```
      A. 散兵志工
      （流量主體）
       ↑       ↓
       └───┬───┘
           │
       C. 打雜社工
        （橋樑）
           │
       B. 小型 NGO
       （核心節點）
```

理由：
- A 提供流量與動能（最未被服務、誘因清楚）
- B 提供結構與持續性（NGO 工作人員是 day-to-day 用戶）
- C 是 A 與 B 之間的天然橋樑（社工既懂民間又有政府身分）
- E（受災民眾）先觀察、待 Basil 帶資料回來再評估如何納入

## 為什麼是現在（Why Now）

3 個趨勢匯流：

1. **2025 花蓮光復暴露結構性缺口**——這次經驗讓 Basil 與其他人有「該做點什麼」的動能
2. **LINE 普及讓「補 LINE 的盲點」變成具體題目**——10 年前還沒有共同的基礎層
3. **AI/LLM 讓「自然語言整合既有資訊流」變得可行**——從一片 LINE 對話抽出結構化需求是 2025 年的技術，不是 2015 年的

**但要小心**：「能做」不等於「該做」。Basil 要避免變成「另一個 Xdite」——做完一次就消散。

## 戰略風險

### 高風險

1. **熱血退散** (D11)：Basil 也是人，第一年熱情過後是否能持續？
2. **使用者無法上來**：花蓮光復現場散兵志工數量不夠 → 沒人用 → 失敗
3. **NGO 不採用**：競爭關係讓 NGO 不願共享資訊 → B 用戶失敗
4. **政治化**：被當成「批判政府的工具」 → 被官方排斥 → 受災民眾不敢用

### 中風險

5. **技術過難**：LLM 整合 LINE 在台灣的可行性未驗證
6. **被收編**：若 NCDR 或政府想合作，會稀釋民間定位
7. **重複 Xdite 失敗**：架完就消散

### 對應策略

| 風險 | 策略 |
|---|---|
| 熱血退散 | 跟早期用戶建立關係，他們也是動能來源 |
| 用戶無法上來 | v1 找一個真實在跑的小型場景測試（不要等大型災害） |
| NGO 不採用 | 從 1-2 個願意合作的小型 NGO 開始，不要找慈濟這種大組織 |
| 政治化 | 偏事實性資訊 (D9)、避免站隊 |
| 技術過難 | v1 先做不需要 LINE 整合的版本 |
| 被收編 | 保持獨立組織形式、不接政府計畫補助 |
| 重複 Xdite 失敗 | 跟 Xdite 本人聯繫學經驗（見 NCDR脈絡與後續行動.md）|

## 給 Basil 寫 vision 的具體建議

### 結構建議

```
1. 我看到了什麼（第一人稱，三次花蓮）
2. 我看到的問題是什麼（連結到問題地圖 P1-P10）
3. 既有方案為什麼填不上（連結到既有方案地圖）
4. ResQ Hub 是什麼（一句話定位 + 拆解）
5. 為什麼是民間（法理空間 + 政府限制）
6. 為什麼是現在（3 個趨勢）
7. v1 是什麼（場景 C+D、用戶 A+B+C）
8. 不是什麼（明確排除）
9. 風險與假設（誠實列出）
10. 接下來 3 個月做什麼
```

### 寫作要點

- **第一人稱寫**：你的三次花蓮經驗是 vision 的 anchor，不是研究綜合
- **North Star 改為結果導向**：不是「建一個 X 工具」，是「達到 Y 結果」
  - 弱版：「建一個物資協作平台」
  - 強版：「散兵志工出發前知道現場最缺什麼，到了以後 5 分鐘內知道該去哪」
- **誠實的負面陳述**：「我們不做的事」會讓 vision 更可信
- **時間維度**：3 個月、1 年、3 年分別會發生什麼
